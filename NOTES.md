# Build Notes

- 2026-07-10: Starting build. Read SPEC.md. Task is large: SvelteKit Pokédex, full toolchain (ultracite+oxlint+oxfmt, lefthook, vitest+Playwright), CI/CD to GitHub Pages, Lighthouse >=90. Working autonomously per anti-stuck rule.
- 2026-07-10: ultracite oxlint config referenced a nonexistent `ultracite/oxlint/sveltekit` preset (only `svelte` exists in this installed version) — fixed. Also disabled a batch of stylistic ultracite rules (sort-keys, func-style, filename-case, etc.) that conflict with idiomatic Svelte 5 runes code and PascalCase component filenames.
- 2026-07-10: Svelte 5 `$effect` dependency tracking bug: reading reactive state only inside a `setTimeout` callback (debounce) does not register it as a dependency — must read it synchronously in the effect body first. Caused search filtering to silently not work.
- 2026-07-10: Guessed GitHub sprites raw-URL path for berries was wrong (`sprites/items/berries/<name>.png` 404s); correct path is `sprites/items/<item-name>.png` (e.g. `cheri-berry.png`), using the berry's `item.name` field from the API.
- 2026-07-10: This sandbox runs multiple sibling agent sessions building similar apps in adjacent directories concurrently. Symptoms seen: (1) a shared /tmp log path got cross-contaminated across sessions producing bizarre package names in dev server logs — fixed by always using a project-local log file; (2) Tailwind utility classes (gap-6, gap-x-8) intermittently failed to generate CSS on a long-lived dev server — resolved by restarting the dev server (stale scan cache), not a real bug; (3) a sibling project's `vite preview` was squatting on the default Playwright port 4173, causing e2e tests to silently run against the wrong app — fixed by pinning a unique, unlikely-to-collide port with `--strictPort`.
- 2026-07-10: GitHub Actions e2e job hung at `Timed out waiting ... from config.webServer` with zero captured output, twice, even after ruling out slow builds by splitting the build into its own visible CI step. Root cause: `vite preview` with no `--host` resolves bind address from the string "localhost", which can land on IPv6-only on the runner while the Playwright client dials `127.0.0.1` explicitly — a connection that hangs with no error surfaced. Fixed by passing `--host 127.0.0.1` explicitly to match the baseURL.
- 2026-07-10: Lighthouse final scores on the live deployed URL: Performance 96, Accessibility 100, Best Practices 100, SEO 100 (all >=90, target met). Key fixes: stable per-slot dex-order rendering (fixed CLS ~1.4->~0), inline aspect-ratio instead of a Tailwind aspect-square class that wasn't being generated in the build, prerendering first-page Pokémon detail so LCP artwork URL ships in the static HTML (LCP 2.6s->1.2s), and darkening type badge colors 45% for WCAG contrast.
